PC VIPR Recon Help
Command Line Format:
   recon_binary -f <raw data filename> [optional switches]

 Basic Usage: 
    -f []                               :[REQUIRED] name of pfile. Can handle repeated scans -f P1.7 -f P2.7
    -out_folder []                      :Output Folder
    -dat_files                          :Export binary files
    -dicom_ideal                        :Export dicom for ideal recon
    -dicom_all                          :Export all 4D flow data as dicoms
    -hdf5                               :Export Images and 4D flow data as HDF5
    -hdf5_only                          :Turn off binary files export
    -hdf5_only                          :Export Images and 4D flow data as HDF5
    -dat_plus_dicom                     :Export binary+Dicom
    -flex                               :Turn on flex processing (requires two echoes)
    -frame_by_frame                     :Recon one frame at a time to reduce memory use
    -combine_echoes                     :Combine all echoes in k-space
    -combine_encodes                    :Combine all encodes in k-space
    -single_encode_tframe               :Uses one encode (1TR of data) instead of five encodes during tframes recon
    -threads []                         :Number of Threads to Use
    -complex_diff                       :Subtract reference encoding
    -flow_recon                         :Flow based recon - only possible with PC data
    -mag                                :Force magnitude only recon
    -data_folder []                     :Location of P-file/gating
    -series_name []                     :Name of series for dicom
    -verbose                            :Verbose output for recon
    -rstart []                          :Start Reciever
    -rend []                            :Stop reciever
    -start_frame []                     :Start frame with frame by frame
    -stop_frame []                      :Stop frame with frame by frame
    -grad_warp                          :Perform gradwarp
    -n4                                 :Perform N4 bias correction on images
    -series_number []                   :Base series for dicom
    -external_timestamps                :Use timing data from alternative file
    -external_timestamps_name []        :File with timing data
    -projection_rejection               :Remove projections based on energy
    -projection_rejection_percentile [] :Threshold of projection energy to remove
    -mod_time []                        :Time in s to synchronize time to (modulo of time vector)
    -max_time []                        :Maximum time (s) to use for imaging

 Geometry Modification: 
    -rcxres []                          :Reconstructed resolution in X
    -rcyres []                          :Reconstructed resolution in Y
    -rczres []                          :Reconstructed resolution in Z
    -zoom_x []                          :Zoom factor in X
    -zoom_y []                          :Zoom factor in X
    -zoom_z []                          :Zoom factor in X
    -oc_shift_off                       :Turn off offcenter shifts

    -extra_shiftx []                    :Spatial shift in mm
    -extra_shifty []                    :Spatial shift in mm
    -extra_shiftz []                    :Spatial shift in mm
    -isotropic_voxels                   :Force isotropic voxels (overrides other geometry mods)
    -k0_correct                         :Align the Phase of the Center of Readout
    -autofov                            :Automatically crop the data to a smaller FOV
    -flip_dicom0 []                     :Flip dicom orientation for direction 0: 0=No flip, 1=Flip, -1=Auto (default)
    -flip_dicom1 []                     :Flip dicom orientation for direction 1: 0=No flip, 1=Flip, -1=Auto (default)
    -flip_dicom2 []                     :Flip dicom orientation for direction 2: 0=No flip, 1=Flip, -1=Auto (default)

    -delayy []                          :Y Gradient delay in us (use with -nogradcal)
    -delayz []                          :Z Gradient delay in us (use with -nogradcal)

    -cal_fsize []                       :Size of filter for calibration
    -no_cal_filter                      :Turn off the calibration filter
    -no_cal_unwrap                      :Don't try to unwrap calibration
    -cal_avg_first                      :Cal: Average first 
    -cal_sub_first                      :Cal: Average first 
    -cal_norm []                        :Norm used for calibration coil comibine (default=2)
    -fcal []                            :name of pfile/scan archive to use for calibration (same size)
    -avg_coil_ringup_correction         :Correct the average coil ringup 
    -full_coil_ringup_correction        :Correct the coil by coil ringup
    -kx_offset []                       :Shift all of Kx to change phase
    -ky_offset []                       :Shift all of Ky to change phase
    -kz_offset []                       :Shift all of Kz to change phase
    -autocal                            :Automatically determine delays using data consistency
    -delay_search                       :Recon a number of delays for search
    -export_gradients                   :Write out the gradient waveforms for each TR

----------------------------------------------
   View Sharing Control 
----------------------------------------------
Usage:
    -viewshare_type []                  :view sharing method tornado/none/hist (defult=none)
                                        :  tornado = variable width filter in kr
                                        :  none = images at equal time intervals, no sharing between frames
                                        :  hist = images with equal data points
    -gating_type []                     :how to gate images
                                        :  resp = respiratory phases
                                        :  ecg = gate by cardiac
                                        :  retro_ecg = retrospective gate by cardiac
                                        :  time = bin by acquisition time
                                        :  prep = bin by time from prep pulses
    -resp_gate []                       :In addition to other gating, perform respiratory gating
                                        :  thresh = threshold respiratory values
                                        :  phase = threshold with upper and lower bounds
                                        :  weight = downweight bad values (see Johnson et al. MRM 67(6):1600
    -resp_gate_signal                   :Specify source for the data used to estimate respiratory phase
                                        :  bellows = signal from respiratory bellows belt in gating file (default)
                                        :  internal = use dc/low spatial frequency information extracted from acquired data
Filter parameters for tornado:
    -vs_wdth_low []                     :width in the center of k-space in frames
    -vs_wdth_high []                    :width in the periphery of k-space in frames
    -vs_vipr_tornado                    :3D radial tornado (default)
    -vs_radial_tornado                  :2D radial tornado
Control for Resp Data
    -correct_resp_drift                 :Median filter with 10s interval
    -resp_gate_efficiency               :Fraction of data to accept
    -resp_sign                          :Flips respiratory waveform
    -resp_phase_lower                   :Lower bound of threshold
    -resp_phase_upper                   :Upper bound of threshold
    -resp_phase_type                    :Phase of respiratory waveform to reconstruct (1=expiration, 2=inspiration, 0=both)
    -resp_gate_weight                   :Soft-Gating Decay Constant (Exponential)
    -adaptive_resp_window               :Length of window to use for thresholding (default=5.0s)
Control for ECG Data

Control for retrospective projection selection
